# Git Rules

## 📦 Git 操作ルール

### コミットメッセージ
- 日本語または英語（Junの好みに合わせる）。
- プレフィックスを付けることを推奨（ `feat:`, `fix:`, `docs:`, `config:` ）。

### ブランチ
- 基本的に `main` で開発。
- 大規模な変更やデバッグは `claude/feature-name` などのブランチを切り、動作確認後にマージ。

### プッシュのタイミング
- デプロイの有無に関わらず、iPhoneから作業を継続したい場合は必ず `git push` を行うこと。
- `deploy.sh` を使用する場合は自動でプッシュされる。
